# 文档 2：产品页面设计（精简版）

## 页面 1：训练安排页

### 页面作用

今日训练总控页。

### 显示内容

- 今日训练归属日
- 今日训练状态
- 计划动作导入入口
- 动作列表

每个动作显示：

- 动作名
- 目标组数
- 组间歇时长
- 已完成组数
- 动作状态
- 剩余倒计时 / 可继续下一组

### 支持操作

- 从计划追加默认动作到今日训练
- 在计划弹层中勾选部分动作后加入今日训练
- 点击动作进入动作页
- 手动新增今日训练动作
- 删除今日训练中未开始动作
- 进入计划编辑页
- 在训练完成后从菜单进入训练总结页

### 页面跳转

- 点击动作 → 动作页
- 点击计划标签 → 打开计划预览，可勾选部分动作后追加到今日训练
- 点击菜单中的计划编辑入口 → 计划编辑页
- 点击菜单中的训练总结入口 → 训练总结页

### 当前补充规则

- 今日训练已完成时，继续追加计划动作前需要确认
- 今日训练已完成时，继续手动新增动作前需要确认
- 当没有计划时，页面仍允许直接手动新增动作
- 计划弹层默认全选全部动作；若一个动作都没选，则禁止提交并提示至少选择 1 个动作

## 页面 2：动作页

### 页面作用

单个动作执行页。

### 显示内容

- 动作名
- 当前进度（已完成组数 / 总组数）
- 动作状态
- 完成本组按钮
- 当前动作剩余倒计时
- 最近一组记录
  - 组序号
  - 完成时间
  - 按动作记录类型展示重量、次数、时长或距离
- 按动作记录类型展示对应补录输入框
- 返回训练安排页

### 支持操作

- 点击“完成本组”
- 手动打开最近一组补录弹层
- 按动作记录类型补录最近一组记录值
- 返回训练安排页

### 页面规则

- 进入页面后，不自动开始当前组计时
- 点击后页面原地更新，不自动跳转
- 已完成动作不可继续点击“完成本组”
- 每个动作页只补录本动作最近一组
- 完成一组后不自动打开最近一组补录弹层
- 若动作使用重量次数记录且来自配置了默认重量/次数的计划，则最近一组补录表单默认带入这些初始值

### 页面跳转

- 返回训练安排页 → 训练安排页

------

## 页面 3：计划编辑页

### 页面作用

长期训练计划维护页。

### 显示内容

- 计划列表
- 当前计划名称
- 计划动作列表

每个动作显示：

- 动作名
- 默认组数
- 默认组间歇时长
- 可选默认重量
- 可选默认次数

### 支持操作

- 新增计划
- 编辑计划名称
- 删除计划
- 新增计划动作
- 编辑计划动作
- 删除计划动作
- 返回训练安排页

### 页面规则

- 这里只维护长期计划
- 不展示训练实时状态
- 不展示倒计时
- 已开始的训练不会因计划编辑自动同步变化

### 页面跳转

- 返回训练安排页 → 训练安排页

### 当前补充规则

- 计划页通过标签切换当前计划
- 计划动作点击进入编辑弹层，左滑可删除，长按打开上下文菜单

------

## 页面 4：训练总结页

### 页面作用

展示本次训练结果。

### 显示内容

- 训练日期
- 当前训练状态
- 完成动作数
- 动作汇总列表

每个动作显示：

- 动作名
- 完成组数
- 每组记录
  - 组序号
  - 完成时间
  - 按动作记录类型展示重量、次数、时长或距离
- 按动作记录类型展示汇总指标

### 支持操作

- 查看总结
- 返回训练安排页

### 当前未展示

- 训练总时长
- 历史训练列表入口
- 动作趋势
- PR 卡片
- 周 / 月总结

### 页面跳转

- 返回训练安排页 → 训练安排页

### 设计图生成 Prompt

```text
为一款移动端健身记录 PWA 设计“训练总结”页面 UI，高保真中文界面，风格简洁、现代、接近 Material 3，但轻量克制。应用名 TrainRe，目标用户是认真记录训练进步的健身用户。

画面尺寸为手机竖屏 390x844。不要做营销落地页，要直接展示真实可用的 App 页面。

项目当前默认浅色主题：
- 页面背景 surface 使用接近米白的 #FBFAF8。
- 卡片和输入容器使用 #F1F0EE / #E8E7E4。
- 主文字和主操作色使用近黑 #202124。
- 次级文字使用 #5F6368。
- 边框使用 #D7D5D2 / #8C8984。
- 状态强调色可以少量使用琥珀色 #7A5B1D。
- 计划或图表辅助色可少量使用蓝 #0B65B9、紫 #7E3FD6、绿 #258A50、青 #007D77、粉 #C74365，但不要让页面变成彩色大屏。

页面结构：
顶部是简洁标题栏，标题“训练总结”，右侧有日历图标按钮。
标题下方是时间范围切换 segmented control：“本周 / 本月”，当前选中“本周”。

第一块是“周总结”概览区：
- 训练 4 天
- 完成 68 组
- 总容量 12,480 kg
- 比上周 +18%
用 2x2 数据网格展示，数字突出，标签小而清晰。

第二块是 PR 卡片：
标题“新纪录”
内容突出一条成就：“卧推估算 1RM 72.5 kg”
副文案：“30 天提升 8.5 kg”
右侧使用奖杯或上升趋势图标，图标使用近黑或琥珀色，不要夸张，不要金色闪光。

第三块是“动作趋势”卡片：
标题“卧推趋势”
显示一条简洁折线图，横轴为最近 6 次训练，纵轴为重量或估算 1RM。
折线从左到右缓慢上升，标出最新点“72.5kg”。
卡片底部显示三个小指标：“最高重量 65kg”、“总容量 3,860kg”、“完成 18 组”。

第四块是“训练分布”：
用简洁横向条形图展示动作模式：
推 42%，拉 31%，腿 19%，核心 8%。

底部保留移动端底部导航栏，包含“今日 / 计划 / 总结 / 设置”，当前高亮“总结”。

视觉要求：
- 整体以米白、浅灰、近黑为主，强调克制和工具感。
- 不要使用绿色或蓝绿色作为主品牌色；当前主色是近黑 #202124。
- 不要使用厚重渐变、紫色主调、橙棕主调、深色主题或高饱和背景。
- 圆角适中，不要过度圆润。
- 信息密度适合健身工具，不要像营销海报。
- 所有文字必须清晰可读，不要出现乱码。
- 不要使用人物照片、肌肉插画、复杂背景、装饰性光效。
- 图表要真实像 App 内组件，而不是数据分析大屏。
- 整体感觉：克制、专业、让用户愿意截图分享。
```
