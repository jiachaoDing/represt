# 分享计划用户路径设计书

## 一句话定位

计划分享不是生成海报，而是让用户用一段文本、一个分享码和一个链接，把可预览、可确认保存的训练计划发给别人。

## 目标

- 让分享者更愿意把训练计划发给朋友、学员或社群。
- 让接收者先看懂计划，再决定是否保存。
- 让新用户在不被强迫下载的情况下体验 RepRest 的计划价值。
- 让安装、添加到主屏幕、保存计划成为计划预览后的自然动作。

## 核心原则

- 先给内容，再给入口。
- 先建立信任，再引导保存。
- 下载入口轻，不遮挡计划查看。
- 分享内容像训练处方，不像营销文案。
- 通过链接或分享码进入后必须能看完整计划。
- App 内只需要输入分享码，不需要扫码能力。
- 保存计划前必须有明确确认。

## 角色

### 分享者

已经在 RepRest 内创建了训练计划，想把计划发给他人。

典型人群：

- 教练给学员发计划。
- 健身搭子互相分享模板。
- 用户把自己的训练安排发到群里。
- 内容创作者发布训练模板。
- 新手向朋友要一份能直接用的计划。

### 接收者

从社交平台、聊天软件或朋友转发中看到计划分享文本。

典型状态：

- 没用过 RepRest。
- 不知道 JSON 是什么。
- 只想先看看计划是否适合自己。
- 愿意保存计划，但不愿意一上来安装 App。

## 分享者路径

### 入口

用户在计划详情或计划列表中点击分享。

可出现的入口：

- 单个计划的更多菜单。
- 训练循环页面的分享入口。
- 设置里的计划分享入口。

设置入口可用于分享一个或多个计划，或分享当前训练循环。

### 分享前预览

用户进入分享预览页或底部面板，看到将要分享的内容。

页面展示：

- 计划名称。
- 动作数量。
- 主要动作。
- 组数和休息摘要。
- 是否包含训练循环。
- 将要发送的分享文本。
- 隐私提示。

隐私提示表达重点：

- 分享后，拿到链接或分享码的人可以查看这份计划。
- 计划名称、动作、组数、休息时间会被展示。
- 分享不包含训练记录、身体数据、历史完成情况。

### 生成分享内容

用户确认后，系统生成一段可复制、可系统分享的文本。

分享文本内容：

- 计划名称。
- 训练摘要。
- 主要动作。
- 分享码。
- Web 导入链接。
- 简短提示：打开链接或在 RepRest 内输入分享码查看并保存计划。

示例：

```text
我分享了一个训练计划：上肢力量 A

包含 6 个动作，约 45 分钟。
主要动作：卧推、划船、肩推、下拉

分享码：R8K4-2P
打开查看并保存：
https://represt.app/share/R8K4-2P
```

### 分享动作

用户可以：

- 调起系统分享。
- 复制分享链接。
- 复制分享码。

第一版优先提供系统分享和复制链接。海报、二维码和保存图片不进入第一版。

## 接收者路径

### 看到分享文本

接收者在聊天、社交平台或群组里看到一段计划分享文本。

接收者首先应该能从文本上判断：

- 这是什么计划。
- 计划大概练什么。
- 是否适合自己。
- 打开链接或输入分享码后能做什么。

### 进入计划预览

接收者点击链接后进入计划预览页；如果已经在 RepRest 内，也可以输入分享码进入同一个预览流程。

这个页面不是传统宣传页，而是计划详情页。

页面打开后直接展示可浏览的计划内容：

- 计划名称。
- 计划摘要。
- 主要动作。
- 训练日或循环信息。
- 主按钮：保存这个计划。

页面上可以有 RepRest 标识，但不能压过计划内容。

### 查看完整计划

接收者可以直接查看完整计划。

完整计划包括：

- 每个训练日。
- 动作名称。
- 目标组数。
- 次数、重量、时长或距离。
- 休息时间。
- 循环安排。
- 备注或说明。

用户不安装也能看完整计划。

### 保存计划

当接收者点击保存时，系统根据环境给出下一步。

已在 RepRest 内：

- 展示导入确认。
- 用户确认后保存到自己的计划库。
- 保存完成后进入计划页。

Android 未安装：

- 提示可以安装 Android App。
- 允许继续使用网页版查看。
- 安装不是查看完整计划的前置条件。

iPhone 用户：

- 提供继续使用网页版。
- 提供添加到主屏幕的引导。
- 保存计划进入 PWA 体验。

桌面浏览器：

- 提供网页版查看。
- 提供复制链接或稍后在手机打开。

内置浏览器：

- 保持可读。
- 在需要保存时提示使用系统浏览器打开。

## 计划预览页结构

### 首屏

目的：让用户快速确认这份计划是否值得保存。

内容：

- 计划名称。
- 分享来源。
- 训练摘要。
- 主要动作。
- 保存按钮。

### 计划详情

目的：让用户完整判断计划质量。

内容：

- 训练日列表。
- 动作列表。
- 组数和目标。
- 休息时间。
- 循环安排。

### 保存区

目的：把“我看懂了”转成“我想保存”。

内容：

- 保存计划按钮。
- 平台相关入口。
- 简短说明保存后可以编辑和记录。

### 轻品牌区

目的：让用户知道工具来源，不抢内容。

内容：

- RepRest 名称。
- 简短产品说明。
- 小入口：了解 RepRest。

## 导入确认路径

用户点击保存后必须进入确认。

确认页展示：

- 将保存的计划名称。
- 动作数量。
- 是否包含循环。
- 主要动作。
- 导入后会创建新的本地计划。

确认页操作：

- 保存。
- 取消。

保存完成后：

- 展示成功反馈。
- 提供查看计划。
- 不自动覆盖现有计划。

## 平台路径

### RepRest 内输入分享码

用户在软件内输入分享码后直接进入计划预览。

路径：

1. 输入分享码。
2. 打开计划预览。
3. 查看计划。
4. 点击保存。
5. 确认导入。
6. 进入计划库。

### Android

用户从链接进入分享页。

路径：

1. 查看计划。
2. 点击保存。
3. 优先在网页版保存到本地计划库。
4. 如果想在 App 内保存，可以复制分享码后在 RepRest 内输入。
5. 安装入口只作为辅助选择，不阻断查看和保存。

### iPhone

用户从链接进入分享页。

路径：

1. 查看计划。
2. 点击保存。
3. 继续使用网页版。
4. 在合适时机引导添加到主屏幕。

### Web/PWA

用户可以在浏览器中完成查看和保存。

路径：

1. 打开分享页。
2. 查看计划。
3. 保存到本地计划库。
4. 后续从 PWA 或浏览器继续使用。

## 分享文本策略

分享文本要让用户愿意转发，也要让接收者不用打开链接就能初步判断计划价值。

应突出：

- 计划名称。
- 训练目标或类型。
- 动作摘要。
- 分享码。
- Web 导入链接。

避免：

- 大面积广告口号。
- 过度品牌露出。
- 需要解释才能看懂的术语。
- 把下载 App 作为主视觉。
- 为第一版引入海报、二维码或扫码权限。

## 转化策略

转化发生在用户看完计划之后。

优先级：

1. 查看完整计划。
2. 保存计划。
3. 继续使用 RepRest。
4. 安装 Android App 或添加 PWA 到主屏幕。
5. 后续在深度使用中引导付费。

分享页不适合强会员墙。

付费更适合出现在：

- 多计划管理。
- 云同步。
- 高级训练分析。
- 高级周期计划。
- 教练和学员管理。
- 品牌化分享物料。

## MVP 范围

第一版支持：

- 分享一个或多个计划。
- 分享循环日程。
- 生成分享文本。
- 分享码查看完整计划。
- 链接查看完整计划。
- 保存前确认。
- Web/PWA 可查看。
- Android 可继续使用 Web，安装入口作为辅助。
- iPhone 可继续使用 PWA。

第一版暂不纳入：

- 公开计划广场。
- 内容推荐流。
- 教练后台。
- 复杂传播统计。
- 卡片海报。
- 二维码。
- App 内扫码。
- 多品牌分享模板。
- 强制登录。
- 打开后自动保存。

## 成功指标

产品侧可以观察：

- 分享按钮点击率。
- 分享创建成功率。
- 分享链接打开率。
- 分享码导入尝试率。
- 计划详情完整查看率。
- 保存计划转化率。
- 新用户保存后次日回访。
- Android 安装转化。
- iPhone PWA 继续使用率。

## 结论

这个功能的用户路径应围绕“计划内容被看见、被理解、被保存”设计。第一版不需要海报、二维码或 App 内扫码；一段清晰文本、一个分享码和一个链接就足够验证计划分享是否成立。真正的增长点不是分享物料本身，而是别人能低摩擦地看到完整计划，并自然进入 RepRest 的使用场景。
