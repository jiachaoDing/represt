# 分享计划卡片用户路径设计书

## 一句话定位

分享计划卡片不是广告页功能，而是让用户把一个可理解、可预览、可保存的训练计划传给他人的轻量分享链路。

## 目标

- 让分享者更愿意把训练计划发给朋友、学员或社群。
- 让接收者先看懂计划，再决定是否保存。
- 让新用户在不被强迫下载的情况下体验 RepRest 的计划价值。
- 让安装、添加到主屏幕、保存计划成为计划预览后的自然动作。

## 核心原则

- 先给内容，再给入口。
- 先建立信任，再引导保存。
- 下载入口轻，不遮挡计划查看。
- 分享卡片像训练处方，不像营销海报。
- 扫码后必须能看完整计划。
- 保存计划前必须有明确确认。

## 角色

### 分享者

已经在 RepRest 内创建了训练计划，想把计划发给他人。

典型人群：

- 教练给学员发计划。
- 健身搭子互相分享模板。
- 用户把自己的训练安排发到群里。
- 内容创作者发布训练模板。
- 新手向朋友要一份能直接用的计划。

### 接收者

从图片、社交平台、聊天软件或朋友转发中看到计划卡片。

典型状态：

- 没用过 RepRest。
- 不知道 JSON 是什么。
- 只想先看看计划是否适合自己。
- 愿意保存计划，但不愿意一上来安装 App。

## 分享者路径

### 入口

用户在计划详情或计划列表中点击分享。

可出现的入口：

- 单个计划的更多菜单。
- 训练循环页面的分享入口。
- 设置里的计划导出入口。

第一版建议优先从单个计划分享开始。

### 分享前预览

用户进入分享预览页或底部面板，看到将要分享的内容。

页面展示：

- 计划名称。
- 动作数量。
- 主要动作。
- 组数和休息摘要。
- 是否包含训练循环。
- 分享卡片预览。
- 隐私提示。

隐私提示表达重点：

- 分享后，看到图片或扫码的人可以查看这份计划。
- 计划名称、动作、组数、休息时间会被展示。
- 分享不包含训练记录、身体数据、历史完成情况。

### 生成卡片

用户确认后，系统生成一张计划卡片。

卡片内容：

- RepRest 标识。
- 计划名称。
- 计划摘要。
- 主要动作。
- 二维码。
- 简短提示：扫码查看并保存计划。

卡片设计气质：

- 像一张清晰的训练处方。
- 品牌露出克制。
- 计划内容优先。
- 二维码清晰可扫。

### 分享动作

用户可以：

- 调起系统分享。
- 保存图片。
- 复制分享链接。

第一版优先提供系统分享和复制链接；保存图片作为可用平台上的补充能力。

## 接收者路径

### 看到计划卡片

接收者在聊天、社交平台、相册或群组里看到图片。

接收者首先应该能从图片上判断：

- 这是什么计划。
- 计划大概练什么。
- 是否适合自己。
- 扫码后能做什么。

### 扫码进入计划预览

接收者扫码后进入计划预览页。

这个页面不是传统宣传页，而是计划详情页。

首屏展示：

- 计划名称。
- 计划摘要。
- 主要动作。
- 训练日或循环信息。
- 主按钮：保存这个计划。
- 次按钮：继续查看完整计划。

页面上可以有 RepRest 标识，但不能压过计划内容。

### 查看完整计划

接收者可以直接查看完整计划。

完整计划包括：

- 每个训练日。
- 动作名称。
- 目标组数。
- 次数、重量、时长或距离。
- 休息时间。
- 循环安排。
- 备注或说明。

用户不安装也能看完整计划。

### 保存计划

当接收者点击保存时，系统根据环境给出下一步。

已在 RepRest 内：

- 展示导入确认。
- 用户确认后保存到自己的计划库。
- 保存完成后进入计划页。

Android 未安装：

- 提示可以安装 Android App。
- 允许继续使用网页版查看。
- 安装不是查看完整计划的前置条件。

iPhone 用户：

- 提供继续使用网页版。
- 提供添加到主屏幕的引导。
- 保存计划进入 PWA 体验。

桌面浏览器：

- 提供网页版查看。
- 提供复制链接或稍后在手机打开。

内置浏览器：

- 保持可读。
- 在需要保存时提示使用系统浏览器打开。

## 计划预览页结构

### 首屏

目的：让用户快速确认这份计划是否值得保存。

内容：

- 计划名称。
- 分享来源。
- 训练摘要。
- 主要动作。
- 保存按钮。

### 计划详情

目的：让用户完整判断计划质量。

内容：

- 训练日列表。
- 动作列表。
- 组数和目标。
- 休息时间。
- 循环安排。

### 保存区

目的：把“我看懂了”转成“我想保存”。

内容：

- 保存计划按钮。
- 平台相关入口。
- 简短说明保存后可以编辑和记录。

### 轻品牌区

目的：让用户知道工具来源，不抢内容。

内容：

- RepRest 名称。
- 简短产品说明。
- 小入口：了解 RepRest。

## 导入确认路径

用户点击保存后必须进入确认。

确认页展示：

- 将保存的计划名称。
- 动作数量。
- 是否包含循环。
- 主要动作。
- 导入后会创建新的本地计划。

确认页操作：

- 保存。
- 取消。

保存完成后：

- 展示成功反馈。
- 提供查看计划。
- 不自动覆盖现有计划。

## 平台路径

### RepRest 内扫码

用户在软件内扫码后直接进入软件内计划预览。

路径：

1. 扫码。
2. 打开计划预览。
3. 查看计划。
4. 点击保存。
5. 确认导入。
6. 进入计划库。

### Android

用户从二维码进入分享页。

路径：

1. 查看计划。
2. 点击保存。
3. 如果已安装，打开 RepRest 保存。
4. 如果未安装，展示安装入口和网页版入口。

### iPhone

用户从二维码进入分享页。

路径：

1. 查看计划。
2. 点击保存。
3. 继续使用网页版。
4. 在合适时机引导添加到主屏幕。

### Web/PWA

用户可以在浏览器中完成查看和保存。

路径：

1. 打开分享页。
2. 查看计划。
3. 保存到本地计划库。
4. 后续从 PWA 或浏览器继续使用。

## 卡片内容策略

卡片要让用户愿意转发。

应突出：

- 计划名称。
- 训练目标或类型。
- 动作摘要。
- 清晰二维码。
- 轻量品牌标识。

避免：

- 大面积广告口号。
- 过度品牌露出。
- 需要解释才能看懂的术语。
- 把下载 App 作为主视觉。

## 转化策略

转化发生在用户看完计划之后。

优先级：

1. 查看完整计划。
2. 保存计划。
3. 继续使用 RepRest。
4. 安装 Android App 或添加 PWA 到主屏幕。
5. 后续在深度使用中引导付费。

分享页不适合强会员墙。

付费更适合出现在：

- 多计划管理。
- 云同步。
- 高级训练分析。
- 高级周期计划。
- 教练和学员管理。
- 品牌化分享卡片。

## MVP 范围

第一版支持：

- 分享单个计划。
- 生成计划卡片。
- 扫码查看完整计划。
- 保存前确认。
- Web/PWA 可查看。
- Android 可引导安装或打开。
- iPhone 可继续使用 PWA。

第一版暂不纳入：

- 公开计划广场。
- 内容推荐流。
- 教练后台。
- 复杂传播统计。
- 多品牌卡片模板。
- 强制登录。
- 扫码后自动保存。

## 成功指标

产品侧可以观察：

- 分享按钮点击率。
- 分享卡片生成率。
- 分享链接打开率。
- 计划详情完整查看率。
- 保存计划转化率。
- 新用户保存后次日回访。
- Android 安装转化。
- iPhone PWA 继续使用率。

## 结论

这个功能的用户路径应围绕“计划内容被看见、被理解、被保存”设计。宣传页只是计划预览页的一部分，下载入口只是保存路径的一部分。真正的增长点不是卡片本身，而是别人扫码后能低摩擦地看到完整计划，并自然进入 RepRest 的使用场景。
