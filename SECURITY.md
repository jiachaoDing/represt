# Security Policy

RepRest currently stores user training data locally and does not require an account.

## Reporting a vulnerability

If you find a security issue, do not open a public issue with exploit details. Contact the maintainer directly and include:

- a short description of the issue
- affected versions or build context
- reproduction steps
- impact estimate

## Scope

Please include only security issues that affect source distribution, build output, local data, sharing flows, or Android packaging.
